# 01. 阶段工程标准与心智模型 (Engineering Standards)

在进入代码的深水区之前，我们需要建立两套核心的“心智模型”。
它们把“普通程序员”和“资深工程师”区分开来。这不仅关乎技术，更关乎**职业习惯**。

---

## 🛡️ 心智模型一：自动化思维 (Automate Everything)

> **原则**: 如果一个命令你需要重复输入三次以上，就把它脚本化。

### 🚨 场景：痛苦的重复劳动

你每天早上开始工作时，都要依次输入：

```bash
cd /project/backend
go run main.go
# (打开新终端)
cd /project/frontend
npm run dev
# (打开浏览器)
open http://localhost:3000
```

这不仅累，而且容易忘或者输错。

### ✅ 解决方案：别名与脚本

#### Level 1: Shell Alias (个人提效)

在你的 `.zshrc` 或 `.bashrc` 里加一行：

```bash
alias dev="cd /project/backend && go run main.go"
```

以后你只用输 `dev`。

#### Level 2: NPM Scripts (团队提效)

在前端项目 `package.json` 里：

```json
{
	"scripts": {
		"dev": "vite",
		"build": "tsc && vite build",
		"deploy": "npm run build && scp -r dist/ user@server:/var/www"
	}
}
```

现在，任何人接手你的项目，只需要知道 `npm run dev` 和 `npm run deploy`，而不需要去记复杂的 SCP 部署命令。**你把经验固化到了代码里。**

---

## 🛡️ 心智模型二：单一数据源 (Single Source of Truth)

> **原则**: 永远不要手动复制文件来“备份”。学会完全信任 Git。

### 🚨 场景：文件名的灾难

你的项目目录里是不是躺着这些文件？

- `app.js`
- `app_backup.js`
- `app_final.js`
- `app_final_fixed.js`
- `app_final_fixed_v2_老板要改的.js`

当你半年后回来，你根本不知道哪个才是最新的，也不敢删任何一个。这就是**数据源污染**。

### ✅ 解决方案：Git 是唯一真理

1.  **想备份？** -> `git commit`。
2.  **想尝试新思路但不确定行不行？** -> `git checkout -b feature/experiment`。
3.  **想找回昨天的代码？** -> `git checkout <commit-hash>`。

保持你的目录极其干净，只有当前正在使用的文件。**历史交给 Git，未来交给分支。**

## 🏆 阶段总结 (Phase Summary)

在阶段零，我们没有写太多复杂的业务代码，但我们磨锋利了手中的刀：

1.  **工具**: 我们掌握了 Git 和 Terminal，不再畏惧命令行。
2.  **网络**: 我们理解了 HTTP 和 DNS，知道了数据是怎么在网线里跑的。
3.  **语言**: 我们扫清了 JS 和 SQL 的语法障碍，准备好构建应用了。
4.  **习惯**: 我们确立了“自动化”和“单一数据源”的工程标准。
